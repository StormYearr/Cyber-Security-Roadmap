## ðŸ“ Key Areas of Defensive Security

### 1. Security Operations Center (SOC)
The SOC is the central team responsible for monitoring a network 24/7. 
* **Focus:** Vulnerabilities, Policy Violations, Unauthorized Activity, and Network Intrusions.
* **Threat Intelligence:** Gathering data about adversaries (Threat Actors) to achieve a threat-informed defense.

### 2. Digital Forensics and Incident Response (DFIR)
* **Digital Forensics:** The science of investigating digital crimes by analyzing File Systems, System Memory, and Logs.
* **Incident Response:** The methodology used to handle a breach.
    * **Phases:** Preparation -> Detection & Analysis -> Containment, Eradication & Recovery -> Post-Incident Activity.

### 3. Malware Analysis
Analyzing malicious software to understand its behavior.
* **Static Analysis:** Inspecting code (Assembly) without executing it.
* **Dynamic Analysis:** Running the malware in a controlled "sandbox" environment to observe its actions.

## ðŸ›  Tool Highlight: SIEM (Security Information and Event Management)
In this room, I interacted with a simulated **SIEM**. A SIEM is the "brain" of a SOC, collecting logs from various sources and presenting them in a single dashboard.

**Key SIEM Capabilities:**
- **Log Aggregation:** Gathering data from firewalls, servers, and databases.
- **Alerting:** Generating notifications when suspicious patterns (e.g., multiple failed logins) are detected.
- **Correlation:** Linking different events to identify a complex attack.
